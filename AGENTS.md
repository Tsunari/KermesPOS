# AGENTS — Repo guidance for AI coding agents

Purpose: give concise, actionable guidance so an AI agent can be immediately productive in this monorepo.

Monorepo layout
- `kermes-pos`: Main POS React app and its local server; primary web UI (see kermes-pos/README.md).
- `kermes-electron`: Electron wrapper that bundles the POS for desktop (Windows packaging with an embedded Python runtime).
- `kermes-web`: Marketing/download site built with Next.js.
- `kermes-menu`: Mobile-first menu site (Next.js).
- `kermes-admin`: Admin dashboard (Next.js).

Quick commands (run from repository root or using `--prefix`)
- Start frontend POS (dev): `npm --prefix kermes-pos run start`
- Start electron (desktop wrapper): `npm --prefix kermes-electron run start`
- Build electron installer: `npm --prefix kermes-electron run build` (uses `electron-builder`, includes `python-3.13.3-embed-amd64` and native modules).
- Next.js apps: `npm --prefix <kermes-web|kermes-menu|kermes-admin> run dev` / `run build` / `run start`.

VS Code tasks
- The workspace defines a compound task `Start POS Dev (compound)` that runs the POS web and electron processes in parallel. Prefer using that for local end-to-end dev.

Docs and tooling
- `typedoc` / `jsdoc` commands exist in several subprojects (see each package.json for `docs` entries).

Important conventions and notes
- Frontend frameworks: `kermes-pos` uses CRA (react-app-rewired). Other web apps use Next.js.
- Packaging: `kermes-electron` packages a Windows installer and ships a Python embed and native modules; be cautious when changing native dependency versions.
- Build artifacts: do not edit files under `*/build` or `*/public` that are generated by the build step. Edit source under `src`.
- Environment: many projects rely on recent Node.js, and native modules (USB/serial) may require Windows dev toolchain when compiling.

What an AI agent should do first
1. Open `AGENTS.md` and the subproject `package.json` files to discover scripts.
2. Use the workspace VS Code tasks for compound runs when verifying changes locally.
3. Link to existing READMEs and docs rather than copy them.

Suggested additional customizations (next steps)

Minimal editing policy
- Keep AI-generated edits confined to the appropriate `src/` folder for the subproject being changed. Run unit/dev builds before proposing packaging changes.
